# 简单模式状态机逻辑分析

## 概述
简单模式是一个二状态机：`idle` → `motion` → `idle`，用于跟踪单向刮刀运动，适用于线性扫描型的加工工艺。

---

## 状态机流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                      开始记录 (recording=True)                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ 状态: idle (默认状态，绿色)                                       │
│ 行为: 等待振动触发                                               │
└──────────────────────────────────────────────────────────────────┘
                              ↓
                    【触发条件】
                magnitude > motion_threshold
                    (振动强度超过阈值)
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ 状态转换: idle → motion (橙色)                                    │
│                                                                  │
│ 【重要操作1】                                                     │
│ 1. 停止上一层的连续采样                                           │
│ 2. 拍摄 BEFORE 图像                                              │
│    - 调用 _capture_before_images()                               │
│    - 记录 CH1、CH2（可见光摄像头）                               │
│    - 记录 CH3（热像仪）                                          │
│    - 触发 move_servo_to_2500() 【开启摄像头】                   │
│ 3. 更新UI状态为 motion                                           │
│ 4. 记录层数到 self.current_layer                                 │
│                                                                  │
│ 【记录操作】                                                      │
│ - _record_main_data(layer, timestamp, "powder_before")           │
└──────────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ 状态: motion (橙色)                                               │
│ 行为: 监测运动是否停止                                            │
└──────────────────────────────────────────────────────────────────┘
                              ↓
            【持续振动】           【振动停止】
        magnitude > threshold    magnitude ≤ threshold
              ↓                            ↓
        更新last_trigger_time        计算停止时间
        继续保持motion状态      time_since_last = current_time - last_trigger_time
                                          ↓
                                  【稳定性检查】
                            time_since_last > debounce_time?
                                   (默认1.0秒)
                            
                            如果 NO → 继续等待
                            如果 YES ↓
┌──────────────────────────────────────────────────────────────────┐
│ 状态转换: motion → idle (绿色)                                    │
│                                                                  │
│ 【重要操作2】                                                     │
│ 1. 拍摄 AFTER 图像                                               │
│    - 调用 _capture_after_images()                                │
│    - 记录 CH1、CH2（可见光摄像头）                               │
│    - 记录 CH3（热像仪）                                          │
│    - 触发 move_servo_to_1500() 【关闭摄像头】                   │
│ 2. 启动当前层的连续采样                                          │
│ 3. 完成层周期                                                     │
│    - 层数 +1                                                      │
│    - 重置状态为 idle                                             │
│                                                                  │
│ 【记录操作】                                                      │
│ - _record_main_data(layer, timestamp, "powder_after")            │
│ - 层数 +1，准备下一层                                            │
└──────────────────────────────────────────────────────────────────┘
                              ↓
                    回到 idle 状态，继续循环
```

## 详细触发时机表

| 事件 | 触发条件 | 操作 | 记录类型 |
|------|---------|------|---------|
| **BEFORE拍摄** | 从idle到motion状态转换 | 停止连续采样 → 拍摄before图像 → 开启摄像头(2500) | `powder_before` |
| **AFTER拍摄** | 运动停止超过debounce_time | 拍摄after图像 → 关闭摄像头(1500) → 启动连续采样 | `powder_after` |
| **层数递增** | AFTER拍摄完成后 | 层数+1，状态重置为idle | - |

---

## 代码执行流程

### 第一步：触发BEFORE拍摄
```
1. _check_motion_state() 检测到 is_triggered=True
   ↓
2. _handle_simple_state_machine(is_triggered=True, magnitude, current_time)
   ↓
3. if self.motion_state == "idle" and is_triggered:
   ├─ self.motion_state = "motion"  # 状态变更
   ├─ self._stop_continuous_sampling()  # 停止上层连续采样
   ├─ self._capture_before_images()  # 拍摄before
   │  └─ _async_capture_images(layer_str, "before", timestamp)
   │     └─ _async_capture_images()中自动调用：
   │        self._record_main_data(layer_str, timestamp, "powder_before")
   └─ 更新UI为motion状态（橙色）
```

### 第二步：监测运动停止
```
1. _check_motion_state() 检测到 is_triggered=False
   ↓
2. _handle_simple_state_machine(is_triggered=False, magnitude, current_time)
   ↓
3. if self.motion_state == "motion" and not is_triggered:
   ├─ time_since_last = current_time - self.last_trigger_time
   └─ if time_since_last > self.debounce_time:  # 默认1.0秒
      ├─ self._capture_after_images()  # 拍摄after
      │  └─ _async_capture_images(layer_str, "after", timestamp)
      │     └─ _async_capture_images()中自动调用：
      │        self._record_main_data(layer_str, timestamp, "powder_after")
      ├─ self._start_continuous_sampling()  # 启动当前层连续采样
      ├─ self._complete_layer_cycle()  # 完成层周期
      │  └─ self.current_layer.set(str(new_layer))  # 层数+1
      │  └─ self.motion_state = "idle"  # 状态重置
      └─ 更新UI为idle状态（绿色）
```

---

## 关键参数

| 参数 | 默认值 | 说明 | 可调整 |
|------|--------|------|--------|
| `motion_threshold` | 0.05 | 振动检测阈值 | ✓ UI滑块/输入框 |
| `debounce_time` | 1.0秒 | 防抖时间（运动停止确认） | ✓ UI输入框 |
| `sampling_interval` | 1.0秒 | 连续采样间隔（对应1Hz） | ✓ UI下拉菜单 |

## main_data记录规则

每次拍摄（before/after）都会记录一条 `main_data` 记录：

```python
record = {
    'layer': layer_num,                    # 层数 (int)
    'timestamp': timestamp,                # 时间戳 (20250919_164834_577)
    'trigger_type': trigger_type,          # "powder_before" 或 "powder_after"
    'image_name_CH1': f"{base_filename}.png",        # 摄像头1图像
    'image_name_CH2': f"{base_filename}.png",        # 摄像头2图像
    'thermal_name_CH3': f"{base_filename}.png",      # 热像仪图像
    'thermal_data_npz': f"{base_filename}.npz",      # 热数据（NPZ格式）
    'thermal_temp_min': thermal_min,       # 热最低温度
    'thermal_temp_max': thermal_max,       # 热最高温度
    'layer_thickness': current_params.get('layer_thickness', 0.0),
    'fill_spacing': current_params.get('fill_spacing', 0.0),
    'fill_speed': current_params.get('fill_speed', 0.0),
    'fill_power': current_params.get('fill_power', 0.0)
}
```

**每层产生 2 条记录：**
- 1 条 `powder_before`（运动开始时）
- 1 条 `powder_after`（运动停止时）

## 舵机控制集成点

### 关闭摄像头（运动开始 - BEFORE阶段）
```python
# 在 _capture_before_images() → _async_capture_images() 中
move_servo_to_2500(servo_id=1, port='COM12')  # 开启摄像头
# 位置2500对应180度（完全打开）
```

### 开启摄像头（运动结束 - AFTER阶段）
```python
# 在 _capture_after_images() → _async_capture_images() 中
move_servo_to_1500(servo_id=1, port='COM12')  # 关闭摄像头
# 位置1500对应90度（中间位置/关闭）
```

## 状态机检查周期

- **检查间隔**：100ms（0.1秒）
- **由 `motion_monitor_timer_id` 控制**
- **只在 `recording=True` 且 `recording_paused=False` 时运行**

```python
if self.recording and not self.recording_paused:
    self.motion_monitor_timer_id = self.frame.after(100, self._check_motion_state)
```

## 与复杂模式的区别

| 特性 | 简单模式 | 复杂模式 |
|------|---------|---------|
| 状态数 | 2个 (idle, motion) | 4个 (idle, first_motion, between_motions, second_motion) |
| 适用场景 | 单向刮刀 | 双向刮刀（往返） |
| 触发条件 | 1次振动 | 2次振动（有间隔） |
| 拍摄次数/层 | 2次 (before + after) | 4次 (before×2 + after×2) |
